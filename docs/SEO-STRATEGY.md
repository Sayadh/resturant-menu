# menus.am — SEO Architecture & Topic Cluster Strategy

> Head-of-SEO blueprint. Target: rank #1 in Armenia (fast wins, low competition)
> and compete internationally with SaaS menu platforms (long-term). Built on the
> Topic Cluster / Pillar-Cluster model, Google 2026 best practices (E-E-A-T,
> helpful-content, entity SEO, semantic search, internal-link equity flow).

---

## 0. Strategic thesis (read first)

1. **Two-market, two-language play.** Armenian content = low competition, fast
   top-3 rankings, builds domain authority cheaply. English content = high
   competition, slow but massive TAM. **Ship Armenian first to earn authority,
   then translate the winners to English.** Every pillar exists in `hy` and `en`.
2. **Pillars are commercial hubs, clusters are informational feeders.** Money
   pages (pillars) rank for head terms; blog articles capture long-tail + funnel
   authority + links up to the pillar.
3. **Entity, not keyword stuffing.** We build menus.am as *the* entity for
   "QR Menu" / "digital restaurant menu" in Google's Knowledge Graph via
   Organization + SoftwareApplication schema, consistent NAP, and topical depth.
4. **Internal links are the growth lever.** 90% of ranking gains after month 3
   come from cluster→pillar link equity and contextual cross-links — not new
   pages.

```

                        │   HOMEPAGE  (menus.am)     │  ← brand + WebSite/Org entity

        ▼              ▼             ▼             ▼              ▼
   ┌─────────┐   ┌──────────┐  ┌──────────┐  ┌──────────┐   ┌──────────────┐
   │ PILLAR  │   │ PILLAR   │  │ PILLAR   │  │ PILLAR   │   │ PILLAR (geo) │

   │         │   │  menu    │  │  menu    │  │  -menu-   │   │  armenia     │
   │         │   │          │  │          │  │  software │   │              │
   └────┬────┘   └────┬─────┘  └────┬─────┘  └────┬─────┘   └──────┬───────┘
        │ 15–20        │             │             │                │
        ▼ clusters     ▼             ▼             ▼                ▼
   ┌─────────────────────────────────────────────────────────────────────┐
   │  SUPPORTING ARTICLES  (/blog/…)  — informational + comparison + how-to │
   │  each links UP to its pillar (keyword anchor) + SIDEWAYS to 2–3 siblings│
   └─────────────────────────────────┬───────────────────────────────────┘
                                      │ every article ends with →
                                      ▼
                    CONVERSION:  Demo  →  Pricing  →  Registration
```

**Rules of the model**
- A cluster article **must** link up to exactly **one** primary pillar with an
  exact/partial-match anchor, plus 2–3 sibling clusters.
- A pillar links **down** to its top 8–12 clusters (contextually, not a dump).
- No orphan pages. Every URL is ≤ 3 clicks from the homepage.

## 5. Internal Linking Strategy

**Link-equity flow (directional rules)**
| From | To | Anchor style | Purpose |
|---|---|---|---|
| Homepage | All 7 pillars | exact-match ("QR Menu", "Online Menu"…) | distribute authority |
| Cluster article | its 1 pillar | exact/partial ("QR Menu platform") | consolidate topic |
| Cluster | 2–3 sibling clusters | descriptive ("how to create a QR menu") | topical depth, dwell |
| Pillar | top 8–12 clusters | partial/long-tail | crawl + relevance |
| Every page | `/restaurant-menu-software` + `/#pricing` | branded/CTA ("menus.am") | conversion |
| Blog index / category | all clusters | title anchors | crawl breadth |

**Anchor-text distribution (avoid over-optimization)**
- 30% exact-match · 30% partial-match · 25% branded/URL · 15% generic ("read
  more", "այս ուղեցույցը"). Never 100% exact — Google penalizes.

**Hierarchy of link priority for a new article**
1. Link **up** to pillar (in intro + conclusion).
2. Link **sideways** to 2 most-related sibling articles (in body).
3. Link **down/out** to the conversion hub + demo (in CTA block).

**Concrete example** — article "QR Menu vs Paper Menu" should:
- link up → `/qr-menu` (anchor "QR Menu platform")
- sideways → "QR Menu advantages", "QR Menu cost"
- convert → "Try the free demo" → `/#demo` → `/#pricing`.

menus.am/en/qr-menu               → English pillar (hreflang pair)

```

**Rules (2026):** flat, lowercase, hyphenated, keyword-in-slug, **no dates**, no
`?id=`, ≤ 60 chars, one canonical per URL. Pillars stay at root (they're products,
not blog posts). Clusters live under `/blog/`. Add `hreflang` `hy` ↔ `en` ↔
`x-default`. Reserve pillar slugs so no restaurant tenant can claim them.

> Do **not** use `/academy/` or `/guides/` for the money pillars — keep them at
> root. `/blog/` is correct for informational clusters (matches Ahrefs/HubSpot).
